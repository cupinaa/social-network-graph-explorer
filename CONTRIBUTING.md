# Contributing

Contributions are welcome.

1. Use Python 3.10 or newer.
2. Keep the project dependency-free unless a new dependency provides clear value.
3. Add or update tests for behavior changes.
4. Run `python -m unittest discover -s tests -v` before submitting a change.
5. Keep user-facing text, identifiers, comments, and documentation in English.

Do not commit generated pickle files, virtual environments, caches, credentials, or private datasets.
